What did I learn?

As the first task in my technical skills course, I completed 12 different GitHub Learning Labs courses. These labs provided an introduction to the fundamental use of GitHub, including:

  • The basics of distributed version control, including creating, reviewing, and approving pull requests
  • A core level of bug tracking, documentation, and source code management throughout the development process
  • The use of HTML and markdown in GitHub
  • Publishing webpages through GitHub Pages

And many other skills.

How is that useful to me?

My target career is as an attorney in the cybersecurity/privacy industry. Although I won’t be developing code on my own, a personal understanding of the process used by developers to generate and moderate code will be key to being able to fluently communicate with other departments in my organization.

By better communicating with the technical team, I will be able to will draft better contracts, better manage employee agreements, and better protect a company’s intellectual property, among other things.

Course Notes

  • Several classmates reported issues with logging in to GitHub or publishing GitHub pages. To be honest, I never experienced these issues.
  • Follow the course instructions. If something seems to be going amiss, go back and check to see if you skipped a step.
  • Markdown is incredibly useful - its how I wrote this page. Try using StackEdit.io or Dillinger to write markdown and see what it will look like it real time.
  • When publishing your resume on GitHub, just do a web search for templates. There are a ton out there, and they are all cloneable projects.
